# 服务器架构与服务分析工具 - 架构设计

## 项目概述
这是一个用于服务器取证和架构分析的工具，专注于分析服务器的服务架构、访问处理流程、重要应用程序、服务器角色以及重点IP关联关系。

## 项目结构
```
server-forensic-tool/
├── src/                  # 主程序源代码
├── modules/              # 功能模块
├── reports/              # 生成的报告输出目录
├── templates/            # 报告模板
├── config/               # 配置文件
└── data/                 # 数据存储
```

## 核心功能模块

### 1. 主程序入口 (src/main.py)
- 命令行界面
- 目标目录输入验证
- 模块调用协调
- 报告输出管理

### 2. 服务配置分析模块 (modules/service_config.py)
- Web服务器配置识别（Apache/Nginx/IIS等）
- 数据库服务配置识别（MySQL/PostgreSQL/MongoDB等）
- 中间件配置识别（Tomcat/JBoss/Redis等）
- 配置文件关键信息提取（监听端口、目录、虚拟主机等）
- 服务启动脚本和配置路径分析

### 3. 网络与端口分析模块 (modules/network.py)
- 监听端口分析（从配置文件和netstat输出）
- 端口与服务映射
- 网络连接分析（ESTABLISHED/LISTENING状态）
- 重点端口识别（Web/DB/SSH等）

### 4. 访问处理流程分析模块 (modules/access_flow.py)
- Web访问日志分析
- 请求路由分析
- 负载均衡配置识别
- 反向代理配置识别
- 访问流量模式分析

### 5. 服务器角色识别模块 (modules/server_role.py)
- Web服务器角色识别
- 数据库服务器角色识别
- 应用服务器角色识别
- 缓存服务器角色识别
- 网关/负载均衡器角色识别
- 混合角色分析

### 6. 重点IP关联分析模块 (modules/ip_analysis.py)
- 访问IP统计与排名
- 内部IP与外部IP分类
- IP与服务的关联关系
- IP访问频率分析
- 可疑IP识别

### 7. 重要应用程序识别模块 (modules/apps.py)
- Web应用程序识别（WordPress/Drupal/自定义应用等）
- 应用程序路径定位
- 应用程序配置文件提取
- 应用程序依赖关系分析

### 8. 报告生成模块 (modules/reporter.py)
- 服务器架构拓扑报告
- 服务依赖关系图
- 访问处理流程图
- IP关联分析报告
- HTML/文本格式报告

## 技术栈
- 编程语言：Python 3.8+
- 核心库：os, pathlib, re, collections, ipaddress
- 可选库：jinja2（HTML报告）
